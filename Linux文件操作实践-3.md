# Linux 实验作业：解剖 C 程序的诞生——深入理解 GCC 编译四大阶段


**实验目的：**
* 掌握 GCC 编译器的基本使用方法。
* 深刻理解 C 语言程序从源代码到可执行文件的四个核心阶段：**预处理（Preprocessing）、编译（Compilation）、汇编（Assembly）、链接（Linking）**。这也能为同学们后续深入研究更高级的编译器优化技术（如 LLVM 架构解析）打下坚实的底层基础。
* 学会查看和分析每个阶段生成的中间文件。

**实验环境：**
* 任意 Linux 发行版（如 Ubuntu, CentOS 等）
* 文本编辑器（Vim, Nano 或 VS Code）
* GCC 编译器已安装（可通过 `gcc --version` 检查）

---

## 实验第一部分：准备源代码

请在你的工作目录下创建一个名为 `hello.c` 的文件，并输入以下代码。这段代码故意包含了宏定义、注释和头文件，以便在后续步骤中观察它们的变化。

```c
// 文件名: hello.c
#include <stdio.h>

#define MULTIPLIER 5

int main() {
    int a = 10;
    /* * 这是一个多行注释，
     * 用于观察预处理阶段的变化。
     */
    int result = a * MULTIPLIER;
    
    printf("The result is: %d\n", result);
    return 0;
}
```

## 实验第二部分：按步骤拆解编译过程

请依次执行以下命令，并在完成每一步后，**务必使用 `cat`、`less` 或 `vim` 打开生成的新文件查看其内容**。

### 阶段一：预处理 (Preprocessing)
预处理器（cpp）负责处理源代码中以 `#` 开头的指令，如展开宏定义、插入头文件内容，并删除所有注释。

* **执行命令：** `gcc -E hello.c -o hello.i`
* **观察任务：** 1. 打开 `hello.i` 文件，翻到最后几行。
    2. 你还能找到 `#define MULTIPLIER 5` 和多行注释吗？
    3. `MULTIPLIER` 变成了什么？代码最前面多出来的几百行代码是什么？

### 阶段二：编译 (Compilation)
编译器（ccl）将预处理后的 `.i` 文件翻译成特定架构的汇编语言代码。

* **执行命令：** `gcc -S hello.i -o hello.s`
* **观察任务：**
    1. 打开 `hello.s` 文件。
    2. 观察里面的内容，你能认出哪些汇编指令？（如 `pushq`, `movl`, `call` 等）。这说明现在的代码已经是机器可以准备理解的底层指令结构了。

### 阶段三：汇编 (Assembly)
汇编器（as）将 `.s` 文件中的汇编指令转换成机器能直接执行的二进制机器码，生成目标文件。对于未来有意向研究二进制逆向工程的同学，这部分目标文件的结构知识尤为关键。

* **执行命令：** `gcc -c hello.s -o hello.o`
* **观察任务：**
    1. 尝试用普通文本编辑器（如 `cat hello.o`）打开它，你会看到什么现象？为什么会这样？
    2. **进阶：** 尝试使用命令 `objdump -d hello.o` 或 `readelf -h hello.o` 来查看这个二进制文件的结构。

### 阶段四：链接 (Linking)
链接器（ld）将你的目标文件（`.o`）与系统标准库（如 C 运行库，里面包含了 `printf` 的真正实现）合并起来，生成最终的可执行文件。

* **执行命令：** `gcc hello.o -o hello_app`
* **观察任务：**
    1. 运行你的程序：`./hello_app`
    2. 使用命令 `ls -l` 查看 `hello.o` 和 `hello_app` 的文件大小，它们的大小差异大吗？为什么？
